# SWE-Master-Coder-3B-SFT (reproducible fork)

A Qwen2.5-Coder-3B-Instruct SFT'd on 56 reward==1 multi-turn agent trajectories generated against SWE-Bench-Verified.

> **This is a demonstration / fork artifact, not a production model.** It was trained as the final closure of an end-to-end SWE-Master pipeline reproduction on a single 8×48GB box. Trained at small scale (56 unique × 5 reps = 280 samples), the model has memorized its training trajectories rather than learning broad SWE generalization. For a production-grade SFT model in this family, see the original [RUC-AIBOX/SWE-Master-32B-SFT](https://huggingface.co/RUC-AIBOX/SWE-Master-32B-SFT) and the rest of the [RUC-AIBOX SWE-Master collection](https://huggingface.co/collections/RUC-AIBOX/swe-agent-series).

## Recipe

| Knob | Value |
|---|---|
| Base model | `Qwen/Qwen2.5-Coder-3B-Instruct` |
| Trainer | [OpenRLHF](https://github.com/OpenRLHF/OpenRLHF) (v0.10.3) `train_sft` |
| ZeRO stage | 3 (sharded across 8 GPUs) |
| Optimizer | AdamW, lr=2e-5, Adam offload to CPU |
| Tricks | flash-attn-2 + Liger Kernel + gradient checkpointing + bf16 |
| Context | 32768 |
| Batch | global=8, micro=1 |
| Epochs | 2 |
| Steps | ~46 optimizer updates |
| Final `train/gpt_loss` | 0.030 |
| Hardware | 8× RTX 6000 Ada (48 GB, no NVLink) |
| Wall time | ~19 min |

## Training data

The 56 trajectories were generated by running R2E-Gym agent rollouts on SWE-Bench-Verified instances, with two teachers:

| Teacher | Phase 1: instances 226–325 | Phase 2: instances 326–425 (mostly) |
|---|---|---|
| [RUC-AIBOX/SWE-Master-32B-SFT](https://huggingface.co/RUC-AIBOX/SWE-Master-32B-SFT) | 23 reward==1 from 100 attempts (23%) | — |
| [Qwen/Qwen3-Coder-30B-A3B-Instruct](https://huggingface.co/Qwen/Qwen3-Coder-30B-A3B-Instruct) | — | 27 reward==1 from 71 attempts (38%) |
| Earlier K=5 + partial K=50 | 6 reward==1 (django subset) | — |
| **Total** | **56 trajectories, 10 repositories** | |

Reward==1 means the agent's patch passed all `FAIL_TO_PASS` tests for the instance.

Repos covered (with reward==1 counts): `django/django` (10), `matplotlib/matplotlib` (5), `pydata/xarray` (6), `psf/requests` (2), `pallets/flask` (1), `scikit-learn/scikit-learn` (15), `pytest-dev/pytest` (7), `sphinx-doc/sphinx` (2), `pylint-dev/pylint` (2), `sympy/sympy` (1).

## Use

```python
from transformers import AutoModelForCausalLM, AutoTokenizer

mid = "nikunjgupta/SWE-Master-Coder-3B-SFT-fork"
tok = AutoTokenizer.from_pretrained(mid)
mdl = AutoModelForCausalLM.from_pretrained(mid, torch_dtype="auto", device_map="auto")

# Wrap into the R2E-Gym openhands scaffold to use as a SWE agent.
# See https://github.com/RUCAIBox/SWE-Master for the agent framework.
```

For agent inference at scale you'd serve this with vLLM:

```bash
vllm serve nikunjgupta/SWE-Master-Coder-3B-SFT-fork \
    --tensor-parallel-size 1 \
    --max-model-len 32768
```

## Honest limitations

1. **Tiny training corpus.** 56 unique trajectories is roughly **1/200th** the size of the SWE-Master paper's SFT dataset (~12,500 trajectories). The model has effectively memorized these examples rather than learning broad SWE patterns.
2. **Trajectory teachers themselves were limited.** The strongest teacher (Qwen3-Coder-30B-A3B-Instruct) achieves ~38% reward==1 on Verified — so half of the training signal comes from a moderately-capable teacher, not a frontier model. The paper used frontier teachers (DeepSeek-V3, GLM-4.6).
3. **No held-out evaluation.** This model was not run against held-out SWE-Bench instances to measure resolve rate. It would almost certainly underperform the [RUC-AIBOX SWE-Master-4B-SFT](https://huggingface.co/RUC-AIBOX/SWE-Master-4B-SFT) — and dramatically underperform the 32B variants.
4. **Trained at 32K context with bf16 cross-entropy** (rather than fp32) — a memory band-aid for 48 GB cards. Numerical stability on the 152K-vocab LM head was traded for memory headroom. May produce slightly less-stable rare-token predictions vs the upstream recipe.

## Reproducing this model

All training/inference code, install scripts, and the 56-trajectory training corpus generation pipeline are at:

**[github.com/Nikunj-Gupta/SWE-Master](https://github.com/Nikunj-Gupta/SWE-Master)** — a fork of [RUCAIBox/SWE-Master](https://github.com/RUCAIBox/SWE-Master).

Run:
```bash
bash install_sft_env.sh
bash install_rollout_env.sh
bash install_vllm_env.sh
bash apply_patches.sh
export WANDB_API_KEY=...
K=100 MAX_WORKERS=16 bash run_full_pipeline.sh
```

See the fork's `SETUP.md` for full setup instructions.
